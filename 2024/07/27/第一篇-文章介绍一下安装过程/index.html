选型问题

在Hexo和Hugo中间选择了Hexo 因为上手简单

Hexo和Hugo都是优秀的静态网站生成器，它们各自有不同的特点和优势。以下是两者的优缺点概述：

Hexo

优点:

  1. 易用性：Hexo的安装和配置过程相对简单，适合初学者。
  2. 主题丰富：Hexo社区提供了大量美观的主题，易于定制和使用。
  3. 插件系统：拥有丰富的插件生态，可以轻松扩展功能。
  4. Markdown支持：原生支持Markdown，方便撰写和发布博客。
  5. 中文支持：在中文用户中较为流行，有较好的中文文档和社区支持。

缺点:

  1. 性能：虽然快速，但可能不如Hugo在构建大型网站时的性能。
  2. 社区活跃度：尽管社区活跃，但可能没有Hugo的社区规模大或活跃。

Hugo

优点:

  1. 速度：Hugo的构建速度极快，非常适合大型网站或博客。
  2. 灵活性：支持多种模板语言（如Go模板），提供更高的灵活性。
  3. 跨平台：在多个操作系统上都能很好地运行。
  4. 安全性：由于模板系统的严格性，Hugo的网站通常更安全。
  5. 社区和文档：拥有一个非常活跃的社区和详尽的文档。

缺点:

  1. 学习曲线：对于初学者来说，模板系统的学习曲线可能稍微陡峭。
  2. 主题和插件：虽然可用，但可能没有Hexo的主题和插件选择多样。

选择建议：

  • 如果你重视速度和灵活性，希望构建大型网站，并且不介意学习一些模板语言，Hugo可能是更好的选择。
  • 如果你希望使用一个易于上手、有大量主题和插件可供选择的系统，并且需要良好的中文支持，Hexo可能更适合你。

最终，选择哪个工具取决于你的个人需求、偏好以及你打算如何使用这个静态网站生成器。两者都是优秀的选择，没有绝对的“好”或“坏”，只有最适合你项目需求的工具。

在 Markdown 文件中，使用以下语法插入图片：

  • alt text: 图片的替代文字，当图片无法显示时会显示这段文字。
  • image-path: 图片的相对路径，从项目根目录开始。比如，如果图片位于 source/images/my-image.png，则路径为 /images/my-image.png。
  • title text: （可选）图片的标题，会在鼠标悬停时显示。
